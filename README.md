# ArtikIO

Онлайн игра в рисование и угадывание слов, аналог Gartic.io

## Описание

ArtikIO - многопользовательская браузерная игра, где игроки по очереди рисуют выбранное слово, а остальные участники пытаются его угадать. Игра поддерживает real-time взаимодействие через WebSocket.

## Основные возможности

### Игровая механика

- Игроки по очереди становятся художниками
- Художник выбирает одно из двух предложенных слов
- Время на выбор слова: 15 секунд
- Время на рисование: 80 секунд
- Остальные игроки пытаются угадать слово через чат
- Первый игрок, набравший 100 очков, побеждает

### Система очков

Для угадывающих игроков:
- Базовые очки: 100
- Множитель времени: чем быстрее угадал, тем больше очков
- Множитель порядка: первый угадавший получает больше очков
- Минимальное количество очков за правильный ответ: 10

Для рисующего игрока:
- 10 очков за каждого игрока, который угадал слово

### Проверка слов

- Точное совпадение: слово угадано
- Однокоренные слова: система показывает "Очень близко!" только угадывающему игроку
- Другие игроки видят вместо близкого слова символы "***"
- Используется морфологический анализ русского языка (pymorphy2)

### Графический редактор

Инструменты рисования:
- Кисть с регулируемым размером (1-50 пикселей)
- Ластик
- Заливка (flood fill алгоритм)
- Линия
- Прямоугольник
- Круг

Дополнительные функции:
- Палитра из 12 предустановленных цветов
- Color picker для выбора любого цвета
- Очистка холста
- Отмена последнего действия

### Темы слов

- Предметы (42 слова)
- Профессии (30 слов)
- Транспорт (29 слов)

Всего: 215 уникальных слов

## Установка

1. Установите зависимости:

```
pip install -r requirements.txt
```

2. Запустите сервер:

```
python main.py
```

3. Браузер откроется автоматически по адресу http://127.0.0.1:5000

## Подключение из локальной сети

Сервер запускается на 0.0.0.0:5000, что позволяет подключаться с любых устройств в локальной сети.

При запуске сервер отображает:
- Локальный IP адрес для подключения из сети (например: http://192.168.1.100:5000)
- Localhost адрес для локального подключения (http://127.0.0.1:5000)

Другие устройства (смартфоны, планшеты, компьютеры) в той же Wi-Fi/Ethernet сети могут подключиться по локальному IP адресу.

Примечание: если брандмауэр Windows блокирует подключения, разрешите входящие соединения для Python на порт 5000

## Архитектура

### Серверная часть

GameManager:
- Управляет всеми игровыми комнатами
- Создает и удаляет комнаты

GameRoom:
- Управляет состоянием одной игровой комнаты
- Хранит список игроков и их очки
- Контролирует раунды и таймеры
- Синхронизирует состояние холста

Player:
- Хранит информацию об игроке
- Отслеживает очки и статус угадывания

### Клиентская часть

GameSocket:
- Управляет WebSocket соединением
- Обрабатывает события от сервера
- Отправляет действия игрока

DrawingCanvas:
- Реализует графический редактор
- Обрабатывает события мыши и touch
- Синхронизирует рисование между клиентами
- Реализует алгоритм заливки

ChatManager:
- Управляет чатом
- Отображает сообщения и системные уведомления

## Протокол взаимодействия

### События клиент -> сервер

- join_game: присоединение к игре
- start_round: начать новый раунд
- word_chosen: выбор слова для рисования
- chat_message: отправка сообщения в чат
- draw_action: действие рисования
- clear_canvas: очистка холста

### События сервер -> клиент

- player_joined: игрок присоединился
- player_left: игрок покинул игру
- choose_word: предложение выбрать слово
- waiting_for_word: ожидание выбора слова
- round_start: начало раунда
- reveal_word: показать слово рисующему
- chat_message: сообщение в чате
- close_guess: близкая попытка угадывания
- player_guessed: игрок угадал слово
- round_end: конец раунда
- game_over: конец игры
- draw_action: действие рисования от другого игрока
- clear_canvas: очистка холста
- sync_game_state: синхронизация состояния для новых игроков
- error: ошибка

## Алгоритмы

### Проверка однокоренных слов

Использует библиотеку pymorphy2 для морфологического анализа:
1. Нормализация слов (приведение к начальной форме)
2. Сравнение нормализованных форм
3. Проверка лексем (все формы слова)
4. Проверка общего корня (первые 4 символа)

### Заливка (Flood Fill)

Реализован алгоритм заливки на основе стека:
1. Получение цвета целевого пикселя
2. Проверка совпадения с цветом заливки
3. Итеративный обход соседних пикселей
4. Использование Set для отслеживания посещенных пикселей

### Расчет очков

Формула для угадавшего игрока:
```
points = BASE_POINTS * time_factor * order_factor
time_factor = max(0, 1 - guess_time / ROUND_TIME)
order_factor = max(0.5, 1 - (guess_order - 1) * 0.1)
points = max(10, points)
```
